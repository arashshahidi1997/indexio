# idea
Since all the subsystems of `projio`: `biblio`, `codio`, `notio` would benefit from a RAG system which is supposed to be provided by `indexio` infrastructure. all also need a chatbot

see
`/storage2/arash/sirocampus` has a chatbot
`/storage2/arash/projects/cogpy` also has a chatbot.

Let's try to unify the chatbot for all the subsystems of `projio` through `indexio` infrastructure.

## Overview
- Unified chatbot added to `indexio` as the `indexio.chat` subpackage
- Any projio subsystem can reuse the same backend + frontend widget
- RAG pipeline: retrieve from indexio vector store, generate via configurable LLM (Ollama or OpenAI-compatible)
- Frontend: reusable JS/CSS widget with persistent localStorage state and markdown rendering

## Implementation

### New files
- `src/indexio/chat/__init__.py` ŌĆö public API
- `src/indexio/chat/models.py` ŌĆö Pydantic `ChatRequest`, `ChatResponse`, `SourceRef`
- `src/indexio/chat/pipeline.py` ŌĆö RAG pipeline (retrieve + LLM generate)
- `src/indexio/chat/settings.py` ŌĆö `ChatSettings` (env-configurable via `INDEXIO_CHAT_*`)
- `src/indexio/chat/app.py` ŌĆö FastAPI app with `/chat/` endpoint + `/health`
- `src/indexio/chat/static/chatbot.js` ŌĆö reusable frontend widget
- `src/indexio/chat/static/chatbot.css` ŌĆö widget styles

### CLI
- `indexio serve` starts the chat server using the default project root and config path

### Install
- `pip install indexio[chat]` pulls in fastapi, uvicorn, pydantic-settings

### Usage in subsystems
Each subsystem's docs site includes the widget:
```html
<script>
  window.INDEXIO_CHAT = {
    apiUrl: "http://localhost:9100/chat/",
    title: "Biblio Assistant",
    storageKey: "biblio_chat_v1",
  };
</script>
<script src="http://localhost:9100/chatbot/chatbot.js"></script>
<link href="http://localhost:9100/chatbot/chatbot.css" rel="stylesheet">
```
